AIRSENTINEL / PUBLIC AIRFLOW INTELLIGENCE
01
Why Now

公共空间的空气问题，不是“有没有新风”，而是“哪里正在积聚”。

局部浓度升高

教室、会议室、车厢等空间会在局部形成 CO2、VOC 或颗粒物积聚，中央系统往往响应慢、颗粒度粗。

人员分布不均

人流密度变化会直接影响空气负荷，传统通风策略很难根据实时占用进行分区调节。

能耗与健康冲突

全量提高通风会增加能耗，而局部不足又影响舒适度、专注力与公共健康安全。

PROBLEM
02
Core Concept

分布式智能空气流通塑造系统

通过传感器网络实时监测 CO2、VOC、PM2.5、温湿度及人流，智能驱动局部风扇、风口和阀门，实现精准分区通风与空气置换。

定位

AIRSENTINEL Public 不是替代中央空调/HVAC，而是作为智能补充系统，专注解决公共空间的局部积聚、响应延迟和分布不均问题。

附加价值

系统可结合屏显与轻微气流进行人流辅助引导，在空气质量、能耗和人员分布之间取得更优平衡。

CONCEPT
03
Operating Loop

感知 → 分析 → 精准控制 → 引导反馈

01 / Sense

多点感知

采集 CO2、TVOC、PM2.5、温湿度、人流与占用状态。

02 / Analyze

边缘分析

ESP32 或 Raspberry Pi 对浓度、趋势、占用进行实时判断。

03 / Control

分区通风

局部风扇、风口和阀门执行脉冲式或定向通风策略。

04 / Guide

人流辅助

通过热力图、大屏和轻微气流提示，帮助人群分散。

OFFICE / CLASSROOM
06
Use Case 02

地铁车厢：优化分层通风，并缓冲高峰密度

空气层面 Airflow

  • 根据车厢区域、人群密度与污染物浓度动态调整局部气流。
  • 减少局部滞留与交叉污染风险，提升高峰期空气交换效率。
  • 为基础新风系统提供更细颗粒度的补充控制。

人流层面 Crowd

  • 通过屏显热力图提示乘客向低密度区域移动。
  • 通过轻微定向气流形成空间提示，减少车厢局部拥挤。
  • 高峰期提供缓冲，但不替代基础运力和基础新风。

VALUE
08
Risks & Response

挑战可控，关键在试点边界与数据透明

成本

从单教室、单车厢、单楼层分阶段试点，验证 ROI 后再规模化降低成本。

维护

建立传感器定期校准和异常漂移提醒机制，避免数据失真。

接受度

人流数据匿名化，仅展示密度与趋势，并用可视化教育用户理解系统价值。

局限

极端高峰仍需基础新风和空间管理配合，系统定位为智能补充而非万能替代。
